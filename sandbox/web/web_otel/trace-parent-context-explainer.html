Explicit Span Parent Context

The bug was not the number of spans. The bug was that some spans were started without the request span as their parent, so Jaeger received several separate traces instead of one request tree.

The screenshot you showed was the left side: substeps appeared as independent search results with only one span each.

The fix changes span topology: scattered roots become children of one request root.

Jaeger can now show one flame/timeline tree because every bar shares the same trace ID and parent chain.

Code References

Before: helper relied on whatever context was active

async function spanStep<T>(name: string, work: () => Promise<T>) {
  return tracer.startActiveSpan(name, async (span) => {
    try {
      return await work();
    } finally {
      span.end();
    }
  });
}

This looked reasonable, but it depended on OpenTelemetry finding the current parent span from async browser context. When that active context was missing, the new span became a new root trace.

After: helper receives the real parent span

async function spanStep<T>(parentSpan: Span, name: string, work: () => Promise<T>): Promise<T> {
  const parentContext = trace.setSpan(context.active(), parentSpan);

  return tracer.startActiveSpan(name, {}, parentContext, async (span) => {
    try {
      return await work();
    } finally {
      span.end();
    }
  });
}

This forces the parent-child link shown in the diagrams. The child span no longer has to guess its parent from the ambient async context.

Before: commit happened after the root workflow returned

async function handleLoad() {
  setStatus("Tracing request...");
  const nextTodo = await loadTodo();

  await spanStep("react.commit_state", async () => {
    setTodo(nextTodo);
    setStatus("Trace exported");
  });
}

This is why react.commit_state appeared as its own one-span trace: it was started after loadTodo had finished, outside the request root span.

After: commit is inside the request root

async function loadTodo(commitState: (todo: TodoViewModel) => void) {
  return tracer.startActiveSpan("request.load_todo", async (span) => {
    await spanStep(span, "ui.validate_intent", async () => {});
    const requestPlan = await spanStep<RequestPlan>(span, "http.prepare_request", async () => {});
    const response = await spanStep<Response>(span, "boundary.browser_to_public_api", async () => {});
    const todo = await spanStep<Todo>(span, "http.parse_json", async () => {});
    await spanStep(span, "react.commit_state", async () => commitState(viewModel));
  });
}

The root span is now passed into every child call, including the React state commit. This produces the right-side Jaeger shape: one root request trace with nested substeps.

Mental model: a trace is not "all spans that happened during the click." A trace is "all spans connected by trace id and parent span id." The fix writes that connection explicitly.
