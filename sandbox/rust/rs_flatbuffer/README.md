# rs_flatbuffer — Inverted Pendulum Simulation

A tutorial project demonstrating **real-time binary telemetry** between a Rust WebSocket backend
and a React frontend using [FlatBuffers](https://google.github.io/flatbuffers/) as the serialisation format.

The simulation is a **cart-pole** (inverted pendulum): a cart on a track with a free-rotating
pole balanced upright by a Linear Quadratic Regulator (LQR) controller running at 60 Hz.

---

## Why FlatBuffers?

At 60 frames per second, serialisation overhead matters.
FlatBuffers produces **zero-copy, zero-parse binary frames** — the receiver reads fields
directly from the raw buffer without allocating any intermediate objects.
Compare this to JSON, which requires string encoding, parsing, and heap allocation on every frame.

## Communication overview

```
   Rust backend                          React frontend
   ────────────                          ──────────────
   Simulation::step()
        │
        ▼
   PendulumState → encode → [binary] ──WebSocket──▶ decode → render canvas
        ▲
        │
   [binary] ◀──WebSocket── encode ◀── slider / click (Parameters / Command)
```

Every WebSocket frame is a two-layer FlatBuffer:

```
Message { type: State | Params | Cmd | Reset, data: [ubyte] }
                                                      │
                                           inner FlatBuffer table
                                           (PendulumState, Parameters, …)
```

The `data` field holds a **second, independently-finished FlatBuffer** nested as raw bytes.
This lets a single WebSocket channel carry multiple message types using the `type` discriminant.

## The schema

`schema/simulation.fbs` is the single source of truth — both sides generate their types from it.

```fbs
enum MessageType : byte { State, Params, Cmd, Reset }

table Message {
  type : MessageType;
  data : [ubyte];        // nested FlatBuffer payload
}

// Backend → Frontend  (60 Hz)
table PendulumState {
  cart_position             : float;
  cart_velocity             : float;
  pendulum_angle            : float;
  pendulum_angular_velocity : float;
  timestamp                 : long;
  is_valid                  : bool = true;
}

// Frontend → Backend  (on slider change)
table Parameters {
  length : float;  mass_cart : float;  mass_pole : float;
  q_pos  : float;  q_vel     : float;  q_ang     : float;
  q_omg  : float;  r_ctrl    : float;
  integration_method : IntegrationMethod = Euler;
}

// Frontend → Backend  (on canvas click)
table Command { target_position : float; }

// Frontend → Backend  (on reset button)
table Reset {}
```

## Running locally

### First time

```sh
task install          # downloads flatc.exe and installs npm deps
task build-schemas    # generates Rust + TypeScript types from the .fbs schema
```

### Development

```sh
task back:run         # cargo run --bin server   →  ws://127.0.0.1:8765
task front:run        # npm run dev              →  http://localhost:5173

# or both concurrently:
task dev
```

### Other tasks

```sh
task back:build       # release build of the Rust backend
task back:run-release # run the release binary
task                  # list all available tasks
```
