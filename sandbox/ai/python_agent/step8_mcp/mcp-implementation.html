Overview

This implementation keeps the original local-tool agent intact and adds MCP as an optional external tool layer.

Piece Responsibility Why it matters
app/main.py Owns CLI args, model calls, message history, and dispatch selection. The agent loop remains easy to read.
app/mcp_client.py Owns MCP server lifecycle and MCP tool conversion. MCP complexity is isolated.
mpc.json Defines the MCP servers loaded at startup. No CLI config flag is needed. 
mcp_config = "mpc.json" if Path("mpc.json").exists() else None

async with MCPToolbox(mcp_config) as mcp_toolbox:
    tools = tool_get_tools() + mcp_toolbox.get_tools()
Design rule: MCP startup is automatic when mpc.json exists. Without that file, only local tools are loaded.

MCP Client Boundary

MCPToolbox is the adapter between this simple agent and external MCP servers.

Field Meaning
exit_stack Keeps stdio streams and client sessions open, then closes them on exit.
sessions Maps server names to initialized ClientSession instances.
tool_to_server Maps model-visible tool names back to the owning MCP server.
tool_specs Stores OpenAI-compatible tool specs generated from MCP metadata. 
params = StdioServerParameters(command=command, args=args, env=env)
read_stream, write_stream = await self.exit_stack.enter_async_context(stdio_client(params))
session = await self.exit_stack.enter_async_context(ClientSession(read_stream, write_stream))
await session.initialize()
Boundary: app/main.py asks for tools and execution; app/mcp_client.py owns MCP protocol details.

Tool Registration

MCP tools are discovered at startup and converted into the same OpenAI function-tool shape used by the existing local tools.

MCP field OpenAI function field
tool.name mcp__server__tool
tool.description function.description
tool.inputSchema function.parameters 
tool_name = f"mcp__{server_name}__{tool.name}"
self.tool_to_server[tool_name.lower()] = server_name
self.tool_specs.append({
    "type": "function",
    "function": {"name": tool_name, "parameters": tool.inputSchema},
})
Consequence: the model sees MCP tools as ordinary function tools, so the main agent loop needs only one dispatch decision.

Runtime Dispatch

At runtime, the model may call either a built-in local tool or a generated MCP tool. 

tool_name = tool_call.function.name
if mcp_toolbox.has_tool(tool_name):
    result = await mcp_toolbox.execute(tool_name, tool_call.function.arguments)
else:
    result = tool_executer(tool_call=tool_call)
Important: MCP execution returns text content that is appended as a standard role=tool message, so the next model call can reason over it.

Configuration and Usage

MCP mode is enabled by passing a config path. Without that flag, the toolbox has zero MCP tools. 

{
  "mcpServers": {
    "example": {
      "command": "uv",
      "args": ["run", "python", "-m", "mcp_servers.example_mcp_server"]
    },
    "random": {
      "command": "uv",
      "args": ["run", "python", "-m", "mcp_servers.random_mcp_server"]
    }
  }
}
Command Behavior
task run -- "hi" Runs the original agent path with local tools.
task run:mcp -- "list files" Runs with servers from mpc.json.
